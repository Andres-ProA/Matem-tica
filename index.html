Relaciones y Funciones: Fundamentos para Programadores

Un vistazo a los conceptos matemáticos detrás del código que escribimos.

1. Relaciones

Una relación es un conjunto de pares ordenados (x, y), donde x pertenece a un conjunto A y y pertenece a un conjunto B. Es una forma de asociar elementos de un conjunto con elementos de otro.

Ejemplo matemático: Considera los conjuntos A = {1, 2, 3} y B = {4, 5, 6}. Una relación R de A a B podría ser R = {(1, 4), (2, 5), (3, 6)}. También podría ser R = {(1, 4), (1, 5), (3, 6)}.

Ejemplo de Software: Relaciones de Base de Datos

En software, una relación es una conexión entre datos, por ejemplo, en una base de datos de una red social. Una relación puede vincular a los usuarios (Conjunto A) con los posts que han creado (Conjunto B). El par ordenado `(usuario_id, post_id)` representa la relación "creado por".

Para crear una relación, haz clic en un usuario y luego en un post. El usuario seleccionado se resaltará.

2. La Función: Una Relación Especial

Una función es una relación con una regla estricta: a cada elemento x de un conjunto A le corresponde exactamente un elemento, llamado f(x), de un conjunto B.

Ejemplo matemático: La regla f(x) = x^2 es una función, ya que para cada valor de x solo hay un único valor de x^2. Por ejemplo, f(2) = 4. Aunque f(-2) = 4 también, cada entrada individual tiene solo una salida.

Ejemplo de Software: Métodos y Funciones

En programación, una función (o método) es un bloque de código que toma una entrada (parámetro) y produce una salida (valor de retorno). Un buen método es determinista: la misma entrada siempre produce la misma salida. Por ejemplo, un método `generarHash(contraseña)` siempre debe devolver el mismo hash para la misma contraseña.

3. El Método

Es útil considerar una función como una máquina. Si x está en el dominio de una función f, entonces cuando x entra a la máquina, es aceptada como entrada y la máquina produce una salida f(x) de acuerdo con la regla de la función. El **Dominio** es el conjunto de todas las posibles entradas, y el **Rango** (o Imagen) es el conjunto de todas las posibles salidas.

**Ejemplo matemático:** Considera la máquina de la función f(x) = 2x. Si la entrada es x = 5, la máquina aplica la regla y la salida es f(5) = 10.

Ejemplo de Software: Parámetros y Retornos

Un método es una máquina que toma un parámetro y devuelve un valor. El parámetro es la entrada (variable independiente) y el valor de retorno es la salida (variable dependiente). Un método como `calcularPrecio(cantidad)` acepta `cantidad` como entrada y devuelve un `precio`, que depende de la cantidad.
